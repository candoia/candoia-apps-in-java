Logic for Null Check Java application:
  This app maintains 3 informations:
      a. total number of revisions
      b. total number of fixing revisions based on commit log
      c. total number of bug fixing revision containing null checking

1. get
    revs: a list of all the revisions and
    issueId: a list of ids of all issues of the projects
2. for every revision in revs{
      a. if revision log contains the pattern of fixing revision then add the revision
         to fixing revision
      b. if revision log is fixing revision and log contains at least a number matching
          any issue id in issueId then add revision in null fixing revision
   }
3. create a chart of the data collect
