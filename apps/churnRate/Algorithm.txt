
Logic for Churn rate Java application:
1. get
    revs: a list of all the revisions and
2. for every revision in revs{
      a. get diffs: A list of all the files changed
      b. compare every file in diffs to its previous version to get number of lines changed.
      c. Add number of lines changed in this revision to number of lines changed till now.
   }
3. for every file divide number of changed lines with number of total revisions.
